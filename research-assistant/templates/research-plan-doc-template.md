# Research Plan Doc Template (Google Docs Format)

This is the canonical structure for the research plan deliverable. Output as markdown that the user pastes into Google Docs (or convert to .docx via the docx skill if requested).

When generating the actual plan, replace bracketed placeholders. Do not include explanatory comments in the final output — those are for the skill, not the reader.

---

# [Project name] — Research Plan

**Author:** [Name] · **Reviewer:** [Name] · **Date:** [YYYY-MM-DD]
**Status:** Draft / Reviewed / Approved
**Linked artifacts:** [Survey Sheet link if applicable] · [Discussion guide link] · [Recruitment brief link]

---

## 1. Research Goal

[One paragraph. Use an outcome-oriented verb: describe, evaluate, identify, measure. Name the audience with behavioural specificity. Name the decision the data will drive.]

**The decision this study will inform:** [What product/business decision rides on this data? If you can't answer, the study isn't ready.]

**Research type:** User research / Market research / Mixed
[State which. If user research, Mom Test rules apply throughout. If market research, stated-preference instruments are valid.]

---

## 2. Research Questions

Primary research question (the one that drives the method choice):
1. [RQ1 — outcome-oriented, falsifiable]

Secondary research questions (nice-to-haves; trim if scope creeps):
2. [RQ2]
3. [RQ3]

Out of scope (explicitly excluded so reviewers don't expand):
- [Topic deliberately not in scope and why]

---

## 3. Method

**Selected method:** [1:1 interview / Moderated UT / In-app survey via Google Forms / Card sort / Tree test]

**Why this method:** [2–3 sentences mapping to the methodology matrix — qual vs quant, attitudinal vs behavioural, lifecycle phase.]

**Methods considered and rejected:** [Briefly — "considered focus group, rejected because of social desirability risk for sensitive trust topic." Helps reviewers see the reasoning.]

**Adjacent methods recommended for follow-up (not planned in detail):**
- [e.g. "A/B test the winning concept post-design — partner with Data Science"]
- [e.g. "Diary study to validate longitudinal habit claims"]

---

## 4. Target Audience

- [Who they are — behavioural, not just demographic]
- [Inclusion criteria — past behaviour, recency, category, market]
- [Exclusion criteria]
- **Sample size:** [N] — [rationale: e.g. "5–8 per segment per Nielsen for qual UT; saturation expected at 6"]

**Secondary segment (if any):**
- [Same structure]
- **Sample size:** [N]

**Total sample:** [N]
**Markets:** [SG / MY / PH / ID / HK / TW — list]
**Languages:** [English / Bahasa / Filipino / Chinese — and which translations needed]

**Recruitment source:** [In-house panel / external partner / CRM]
**Incentive:** [Amount + rationale per market]
**Quota controls:** [e.g. "50% buyer-led, 50% seller-led; max 30% Pro Sellers"]

---

## 6. Discussion / Question Guide

[Embed the full guide here. Pull from the appropriate template:
- 1:1 interview / UT → use interview-guide-section.md structure (warm-up / past-behaviour core / emotional deep-dive / wrap)
- Card sort / tree test → use card-sort-tree-test-protocol.md
- Survey → put question summary here, link to the Google Sheet]

For survey studies, the doc says:

> **Survey instrument:** see linked Google Sheet → [link]
> The Sheet is the source of truth. Each row = one question with type, options, branching, and objective mapping. The Google Form is built from that Sheet.

---

## 7. Pre-Fielding Stakeholder Briefing (Assumptions To Externalise)

Before fielding, the team must list:
- What we believe is true today (and why)
- What we hope to confirm
- What we are afraid to find
- What outcomes would change the roadmap

[Capture this collaboratively. The doc is opinionated about this — externalising assumptions reduces confirmation bias during fielding and synthesis.]

---

## 8. Risks and Limitations

- **Bias risks specific to this method:** [Hawthorne for lab UT, social desirability for self-report, etc.]
- **Sample limitations:** [Convenience sampling, regional skew, language coverage]
- **What this study cannot answer:** [Be explicit about scope]
- **Mitigations in place:** [Behavioural anchoring, contextual inquiry, blind synthesis, etc.]
